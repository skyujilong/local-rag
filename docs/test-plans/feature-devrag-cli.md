## 1. 测试概述

### 1.1 测试目标

**主要目标**
- 验证 devrag-cli MVP 功能的完整性和正确性
- 确保系统满足性能和非功能需求
- 发现并修复关键缺陷，保证产品质量
- 为用户发布提供质量保障

**测试范围**
- MVP v1.0 核心功能（9 个功能模块）
- 性能和可靠性测试
- 安全和隐私测试
- 集成测试（API、MCP、第三方服务）

**测试不包含**
- 团队协作功能（v1.2）
- 云端同步功能（v2.0）
- 移动端适配
- 多用户权限管理

### 1.2 测试策略

**测试类型**
1. **功能测试**：验证每个功能模块的业务逻辑
2. **性能测试**：验证响应时间、吞吐量、资源占用
3. **安全测试**：验证数据隐私、访问控制、输入验证
4. **集成测试**：验证组件间交互和第三方服务集成
5. **用户体验测试**：验证界面易用性和操作流程

**测试方法**
- 手工测试：探索性测试、UI 测试
- 自动化测试：API 测试、性能测试
- 灰盒测试：结合代码分析和功能测试

**测试优先级**
- **P0（阻塞）**：核心功能失败，必须修复
- **P1（严重）**：主要功能受损，严重影响使用
- **P2（一般）**：次要功能问题，有变通方案
- **P3（轻微）**：UI 问题、文案错误

### 1.3 测试环境

**硬件环境**
- CPU：Apple M1/M2 或 Intel x64
- 内存：8GB+（推荐 16GB）
- 磁盘：20GB 可用空间

**软件环境**
- 操作系统：macOS 12+, Ubuntu 20.04+, WSL2
- Node.js：v18.17.0+（推荐 v20 LTS）
- Ollama：v0.1.0+（运行 nomic-embed-text 模型）
- 浏览器：Chrome 90+, Firefox 88+, Safari 14+

**测试数据**
- 正常数据：标准 Markdown 文件、普通网页 URL
- 边界数据：大文件（10MB+）、特殊字符、空文件
- 异常数据：非法 URL、损坏文件、超长文本
