## 2. 测试策略

### 2.1 测试类型

#### 单元测试
- **目标**：验证独立模块功能的正确性
- **覆盖范围**：
  - 编辑器组件渲染逻辑
  - 自动保存定时器逻辑
  - 向量化触发机制
  - 标签管理数据操作
  - 文档分块算法
  - 混合查询策略（降级判断、结果合并）

#### 集成测试
- **目标**：验证模块间协作的正确性
- **覆盖范围**：
  - 编辑器 → API → 文件系统 → 向量化流程
  - 标签筛选 → ChromaDB 预过滤 → 向量搜索流程
  - 混合查询两阶段执行流程
  - MCP 接口 → 数据层交互
  - 自动保存 → LocalStorage → 服务器同步

#### E2E 测试（端到端测试）
- **目标**：验证用户完整操作流程
- **覆盖范围**：
  - 创建笔记 → 编辑 → 自动保存 → 向量化 → 搜索
  - 标签创建 → 笔记打标签 → 标签筛选 → 结果验证
  - 混合查询场景（标签准确、标签不准确、无标签）

#### 性能测试
- **目标**：验证系统性能指标达标
- **关键指标**：
  - 编辑器加载时间 < 1 秒
  - 保存响应时间 < 500ms
  - 向量化延迟 < 3 秒
  - 混合查询性能（阶段1 ~50ms，阶段1+2 ~550ms）
  - 编辑器内存占用 < 50MB

#### 兼容性测试
- **目标**：验证跨平台兼容性
- **测试环境**：
  - 浏览器：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
  - 移动端：iOS Safari 14+、Android Chrome 90+
  - 屏幕尺寸：最小宽度 320px

### 2.2 测试优先级

| 优先级 | 说明 | 功能示例 |
|--------|------|---------|
| P0 | 核心功能，阻塞发布 | 创建笔记、保存、向量化、搜索 |
| P1 | 重要功能，影响体验 | 标签管理、自动保存、MCP 接口 |
| P2 | 增强功能，可延后 | 工具栏、快捷键、主题切换 |

### 2.3 测试工具

- **单元测试**：Vitest
- **集成测试**：Vitest + MSW（Mock Service Worker）
- **E2E 测试**：Playwright
- **性能测试**：Lighthouse、Chrome DevTools
- **API 测试**：Postman / curl
