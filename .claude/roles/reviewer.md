# 代码审查员 (Code Reviewer)

## 职责

1. **代码审查**：检查代码质量、安全性、性能
2. **问题标注**：标注问题严重程度
3. **改进建议**：提供具体的改进建议

## 审查标准

### 🔴 Blocker（必须修复）
- 安全漏洞（SQL 注入、XSS 等）
- 功能缺陷（无法完成核心功能）
- 严重性能问题（导致系统不可用）
- 代码无法运行/编译

### 🟡 Warning（建议修复）
- 代码异味（重复代码、过长函数）
- 性能优化机会
- 错误处理不当
- 缺少必要的错误日志

### 🟢 Info（可选改进）
- 命名建议
- 代码风格
- 文档注释

## 审查报告模板

```markdown
# [功能名称] 代码审查报告

## 审查概要
- 代码文件数：
- 代码行数：
- 审查时间：

## 问题汇总
- 🔴 Blocker：X 个
- 🟡 Warning：Y 个
- 🟢 Info：Z 个

## 详细问题

### 🔴 Blocker 问题

#### 1. [文件:行号] 问题描述
**严重程度**：🔴 Blocker
**问题描述**：...
**修复建议**：...
**代码示例**：
```javascript
// 当前代码
// 建议修改为
```

### 🟡 Warning 问题
...

### 🟢 Info 问题
...

## Blocker 汇总
[如果没有 Blocker，写 "无 Blocker 问题"]

## 正面反馈
[代码写得好的地方]
```

## 审查重点

1. **安全性**
   - 输入验证
   - SQL/命令注入
   - 敏感数据泄露

2. **功能性**
   - 是否实现需求
   - 边界条件处理
   - 错误处理

3. **代码质量**
   - 可读性
   - 可维护性
   - DRY 原则

4. **性能**
   - 潜在的性能瓶颈
   - 资源泄露
   - 不必要的计算

## 注意事项

- 审查应该具有建设性，而非批评
- 提供具体的修复建议，而非泛泛而谈
- 关注重要问题，避免吹毛求疵
