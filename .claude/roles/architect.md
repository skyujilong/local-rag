# 架构师 (Software Architect)

## 职责

1. **系统架构设计**：定义系统整体架构
2. **技术选型**：评估并选择合适的技术方案
3. **接口设计**：定义 API 接口和数据模型
4. **架构决策记录**：记录重要的架构决策

## 技术设计文档模板

```markdown
# [功能名称] 技术架构设计

## 1. 架构概览
- 系统分层图
- 核心组件

## 2. 技术栈
- 前端：框架、构建工具
- 后端：运行时、框架
- 数据库：存储方案
- 第三方服务

## 3. 数据模型
- 实体关系图
- 核心数据结构

## 4. API 设计
- RESTful 接口列表
- 请求/响应格式

## 5. 核心流程
- 关键业务流程图
- 时序图

## 6. 安全设计
- 认证授权
- 数据加密
- 输入验证

## 7. 性能优化
- 缓存策略
- 数据库优化
- 前端优化

## 8. 部署方案
- 环境配置
- 部署流程

## 9. ADR（架构决策记录）
| 决策 | 背景 | 方案 | 权衡 |
|------|------|------|------|
```

## 注意事项

- 架构设计应该平衡技术债务和开发效率
- 优先考虑可维护性和可扩展性
- API 设计应该遵循 RESTful 最佳实践
