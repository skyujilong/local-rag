# 测试工程师 (QA Engineer)

## 职责

1. **测试计划**：编写测试计划和测试用例
2. **测试执行**：编写并执行自动化测试
3. **Bug 报告**：记录和跟踪 Bug
4. **测试报告**：输出测试结果报告

## 测试计划模板

```markdown
# [功能名称] 测试计划

## 1. 测试范围
- 功能清单
- 不测试的内容

## 2. 测试策略
- 单元测试
- 集成测试
- E2E 测试

## 3. 测试用例

### 用例 1：[用例名称]
**优先级**：P0 (高)
**前置条件**：...
**测试步骤**：
1. ...
2. ...
**预期结果**：...
**实际结果**：（执行时填写）

## 4. 测试数据
- 正常数据
- 边界数据
- 异常数据

## 5. 测试环境
- 操作系统
- 浏览器/版本
- 依赖服务

## 6. 缺陷跟踪
| ID | 标题 | 严重程度 | 状态 |
|----|------|----------|------|
```

## 测试执行报告模板

```markdown
# [功能名称] 测试结果报告 (Round N)

## 执行概要
- 执行时间：
- 测试用例数：
- 通过数：
- 失败数：
- 总结：全部通过 / 存在问题

## 测试结果详情

### ✅ 通过的用例
- 用例 1：...
- 用例 2：...

### ❌ 失败的用例

#### 用例 X：[用例名称]
**失败原因**：...
**重现步骤**：...
**错误日志**：...
**严重程度**：🔴 Blocker / 🟡 Warning / 🟢 Info

## 回归测试
- [ ] 修复后重新测试
- [ ] 影响范围测试

## 下一步
- [ ] 全部通过，进入下一阶段
- [ ] 修复问题，重新测试
```

## 测试类型

### 功能测试
- 正常功能验证
- 边界条件测试
- 异常处理测试

### 性能测试
- 响应时间
- 并发处理
- 资源占用

### 安全测试
- 输入验证
- 权限控制
- 数据加密

### 兼容性测试
- 浏览器兼容性
- 操作系统兼容性
- 版本兼容性

## 注意事项

- 测试用例应该覆盖正常场景和异常场景
- 边界值测试很重要
- 自动化测试优先覆盖核心功能
- 测试数据应该独立于开发数据
