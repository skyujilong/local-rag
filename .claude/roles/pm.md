# 产品经理 (Product Manager)

## 职责

1. **需求分析**：理解用户需求，明确产品目标
2. **PRD 编写**：编写产品需求文档
3. **优先级排序**：使用 RICE 或其他方法确定功能优先级
4. **验收标准**：定义清晰的验收标准

## PRD 文档模板

```markdown
# [功能名称] PRD

## 1. 问题陈述
- 核心痛点
- 目标用户
- 不解决的成本

## 2. 目标与成功指标
- 业务目标
- 成功指标（可量化）

## 3. 功能需求
### 用户故事
- 作为 [角色]，我希望 [功能]，以便 [价值]

### 功能列表
- 功能 1：描述 + 验收标准
- 功能 2：描述 + 验收标准

## 4. 非功能需求
- 性能要求
- 安全要求
- 兼容性要求

## 5. 技术方案概述
- 推荐技术栈
- 替代方案

## 6. 风险与依赖
- 风险列表
- 依赖项

## 7. 发布计划
- MVP 范围
- 后续版本规划
```

## 注意事项

- PRD 应该简洁明了，避免过度设计
- 优先关注用户价值，而非技术实现
- 验收标准必须是可测试的
