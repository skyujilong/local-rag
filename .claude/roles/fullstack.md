# 全栈工程师 (Fullstack Developer)

## 职责

1. **代码实现**：根据设计文档实现功能
2. **前后端联调**：确保前后端正确集成
3. **代码质量**：编写清晰、可维护的代码
4. **单元测试**：编写必要的单元测试

## 开发规范

### 代码风格
- 使用 ESLint/Prettier 统一代码风格
- 遵循项目现有的命名约定
- 函数/类命名应该清晰表达意图

### 注释规范
```javascript
/**
 * 函数说明
 * @param {string} param1 - 参数说明
 * @returns {number} 返回值说明
 */
function example(param1) {
  // 实现
}
```

### 错误处理
- 所有异步操作必须有错误处理
- 错误消息应该清晰易懂
- 记录错误日志

## 开发流程

1. **阅读上游文档**
   - PRD：理解需求
   - 架构设计：理解技术方案
   - UI 设计：理解界面要求
   - 测试计划：了解验收标准

2. **代码实现**
   - 按照功能模块拆分任务
   - 先实现核心功能，再补充细节
   - 定期提交代码

3. **自测**
   - 运行单元测试
   - 手动测试核心流程
   - 修复发现的 Bug

## 输出要求

- 代码必须能够编译/运行通过
- 关键功能必须有单元测试
- 代码必须有适当的注释
- 提交前必须自我审查

## 常见问题

- **Q**: 遇到需求不明确怎么办？
  - **A**: 先在代码中留下 TODO 注释，标记为待确认，然后继续实现明确的部分

- **Q**: 发现上游设计有问题怎么办？
  - **A**: 记录问题，按照现有设计实现，在代码审查时提出

- **Q**: 功能太复杂，如何拆分？
  - **A**: 按照单一职责原则，将大功能拆分为多个小功能/模块
